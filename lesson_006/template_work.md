# Последовательность команд при стандартной работе
 1. Инициализация репозитория (git init)
 2. Создание папок и файлов (mkdir, echo)
 3. Коммит изменений (git add -> git commit)
 4. Создание новой ветки (git branch)
 5. Внесение изменений в новую ветку (mkdir, echo)
 6. Коммит изменений в новой ветке(git add -> git commit)
 7. Переключение на основную ветку(git switch)
 8. Слияние веток(git merge)
 9. Удаление вспомогательное ветки(git branch -d)
